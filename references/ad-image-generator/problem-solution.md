# Problem-Solution 广告 Prompt 架构设计心法

> 核心理念：**先让用户感受痛苦，再展示解脱**

## 一、策略 定位

Problem-Solution 类型广告的核心目标是**通过对比"痛点"和"解决方案"来创造购买动机**。用户必须先认同问题的存在，才会渴望解决方案。

**关键心智模型**：
- 问题 → 用户的痛点共鸣
- 产品 → 痛点的终结者
- 对比 → 购买动机的催化剂

**情感曲线**：
```
不适/挫败 → 紧张/期待 → 解脱/满足
(Problem)    (Tension)    (Solution)
```

## 二、标准章节结构

### 2.1 核心章节（必选）

| 章节 | 出现率 | 核心作用 |
|------|--------|----------|
| COMPOSITION NARRATIVE | 高 | 问题→解决的视觉叙事 |
| UNEXPECTED ELEMENT | 高 | 创造视觉转折点 |
| VISUAL DISTINCTIVENESS | 40% | 对比的记忆点 |
| HEADLINE TREATMENT | 40% | 问题-解决式文案 |
| PRODUCT PRESENTATION | 40% | 产品作为"救星" |

### 2.2 特色章节（Problem-Solution 独有）

| 章节 | 核心作用 |
|------|----------|
| PROBLEM ZONE | 痛点场景展示 |
| SOLUTION ZONE | 解决方案展示 |
| VISUAL PIVOT | 视觉转折点 |
| EMOTIONAL ARC | 情感变化曲线 |

## 三、章节设计套路

### 3.1 COMPOSITION NARRATIVE（构图叙事）

**设计原则**：用空间讲述"从问题到解决"的故事

**常用套路**：

1. **垂直分割叙事**
   ```
   上部 30%：问题场景（冷色调、混乱、不适）
   中部 25%：产品作为转折点（对角线位置）
   下部 35%：解决场景（暖色调、清晰、舒适）
   底部 10%：CTA
   → 眼球自然从问题流向解决方案
   ```

2. **左右对比叙事**
   ```
   左侧：问题状态（灰暗、复杂、挫败）
   中心：产品作为分割线
   右侧：解决状态（明亮、简洁、满足）
   → 直观的前后对比
   ```

3. **对角线转折叙事**
   ```
   从左上（问题）到右下（解决）的对角线流动
   产品位于对角线中心，成为"转折点"
   → 动态的问题解决过程
   ```

**示例模板**：
```
COMPOSITION NARRATIVE:
- Structure: [Vertical/Horizontal/Diagonal] split
- Problem zone (top/left): [problem visualization]
- Pivot point (center): Product as the [bridge/solution/breakthrough]
- Solution zone (bottom/right): [solution visualization]
- Eye flow: Problem → Product → Solution → CTA
```

### 3.2 PROBLEM ZONE（问题区域）

**设计原则**：让用户"感受到"问题，而非只是"看到"问题

**视觉化痛点套路**：

1. **温度流失可视化**
   ```
   蒸汽/热量正在逃逸
   颜色从暖变冷
   → "温度在流逝"的视觉隐喻
   ```

2. **混乱/复杂可视化**
   ```
   多个物品散乱（旧保温杯、清洁剂、毛巾）
   视觉上的"杂乱"
   → "麻烦"的视觉表达
   ```

3. **挫败情绪可视化**
   ```
   人物表情或肢体语言暗示不满
   或通过色调（灰暗、冷色）传递情绪
   → 情感层面的痛点共鸣
   ```

4. **时间压力可视化**
   ```
   时钟、温度计显示"下降"
   → 紧迫感和焦虑
   ```

**示例模板**：
```
PROBLEM ZONE:
- Visual metaphor: [specific pain point visualization]
- Color tone: [cool/grey/muted] suggesting [discomfort/frustration]
- Elements: [specific items that represent the problem]
- Emotion: Viewer should feel [specific negative emotion]
- Text overlay (optional): "[痛点描述]" in fading typography
```

### 3.3 UNEXPECTED ELEMENT / VISUAL PIVOT（视觉转折点）

**设计原则**：产品必须"打断"问题，成为戏剧性的转折

**常用套路**：

1. **对角线切入**
   ```
   产品以35-45度角"切入"画面
   打破问题区域和解决区域的边界
   → 产品是"破局者"
   ```

2. **光芒/能量爆发**
   ```
   产品周围有光晕或能量效果
   从产品向外辐射到解决区域
   → 产品是"能量源"
   ```

3. **内部揭示**
   ```
   产品剖面图或透视图
   展示内部技术（保温层、滤网）
   → 技术是解决问题的关键
   ```

4. **桥梁隐喻**
   ```
   产品位于问题和解决之间
   视觉上"连接"两个世界
   → 产品是"桥梁"
   ```

**示例模板**：
```
UNEXPECTED ELEMENT - THE VISUAL PIVOT:
- Product position: [diagonal/center/bridging]
- Visual treatment: [glow/cross-section/energy burst]
- Role: Product [interrupts/bridges/transforms] the problem
- This is the "aha moment" - where problem meets solution
```

### 3.4 SOLUTION ZONE（解决区域）

**设计原则**：展示"问题解决后"的理想状态

**视觉化解决套路**：

1. **温度恒定可视化**
   ```
   蒸汽稳定、温度计显示恒定
   暖色调、舒适感
   → "温度保持"的视觉证明
   ```

2. **简洁/有序可视化**
   ```
   只有产品，干净整洁
   视觉上的"简单"
   → "便捷"的视觉表达
   ```

3. **满足情绪可视化**
   ```
   人物享受使用产品的瞬间
   或通过色调（温暖、明亮）传递情绪
   → 情感层面的满足感
   ```

4. **自由/轻松可视化**
   ```
   户外场景、自然光线
   产品融入美好生活
   → 问题解决后的自由感
   ```

**示例模板**：
```
SOLUTION ZONE:
- Visual metaphor: [specific solution visualization]
- Color tone: [warm/bright/clear] suggesting [comfort/satisfaction]
- Elements: [specific items that represent the solution]
- Emotion: Viewer should feel [specific positive emotion]
- Product shown: [in use/in ideal context]
```

### 3.5 HEADLINE TREATMENT（标题处理）

**设计原则**：文案必须呼应"问题→解决"的叙事

**常用文案套路**：

1. **对比式**
   ```
   "温度在流逝" → "温度恒定"
   "旧的麻烦" → "新的自由"
   → 问题和解决的直接对比
   ```

2. **否定式**
   ```
   "不再担心温度下降"
   "告别保温焦虑"
   → 强调问题的消除
   ```

3. **承诺式**
   ```
   "24小时，温度只降2度"
   "从此，热饮永驻"
   → 解决方案的具体承诺
   ```

4. **疑问式**
   ```
   "你试过24小时还在冒热气的杯子吗？"
   → 引发好奇，暗示解决方案存在
   ```

**示例模板**：
```
HEADLINE TREATMENT:
- Problem headline (optional): "[痛点描述]" - fading/muted
- Solution headline: "[解决承诺]" - bold, prominent
- Position: [integrated with composition flow]
- Typography: Hanken Grotesk, contrast between problem/solution text
- Brand color #a36710 for solution emphasis
```

### 3.6 EMOTIONAL ARC（情感弧线）

**设计原则**：设计用户的情感旅程

**标准情感弧线**：

```
1. 不适/共鸣 (Problem Zone)
   ↓
2. 紧张/期待 (Visual Pivot)
   ↓
3. 解脱/满足 (Solution Zone)
   ↓
4. 行动/渴望 (CTA)
```

**示例模板**：
```
EMOTIONAL ARC:
- Stage 1 (Problem): Viewer feels [discomfort/frustration/recognition]
- Stage 2 (Pivot): Viewer feels [curiosity/hope/anticipation]
- Stage 3 (Solution): Viewer feels [relief/satisfaction/desire]
- Stage 4 (CTA): Viewer feels [urgency/motivation to act]
```

## 四、问题-解决对照表

| 问题（痛点） | 视觉表达 | 解决（卖点） | 视觉表达 |
|-------------|----------|-------------|----------|
| 温度下降 | 蒸汽逃逸、冷色调 | 24小时保温 | 蒸汽稳定、暖色调 |
| 清洁麻烦 | 清洁剂、刷子散乱 | 易清洁 | 干净表面、光泽 |
| 细菌滋生 | 暗示性的不洁感 | 抗菌材料 | 光晕、洁净感 |
| 无法泡茶 | 茶叶散乱、不便 | 内置滤网 | 茶叶优雅浸泡 |
| 携带不便 | 笨重、累赘 | 便携设计 | 轻松手持、户外场景 |

## 六、与其他 策略 的选择指南

| 用户状态 | 推荐 策略 |
|----------|-------------|
| 不知道有问题 | Problem-Solution（唤醒痛点） |
| 知道问题，不知道解决方案 | **Problem-Solution** |
| 知道解决方案，比较产品 | Features-Benefits |
| 已有偏好，需要情感连接 | Lifestyle / Testimonial |

---

## 七、避免的常见错误

1. **问题不够痛** → 痛点必须让用户"感同身受"
2. **转折不够戏剧** → 产品必须是"英雄式"登场
3. **解决不够爽** → 解决区域必须让用户"渴望"
4. **对比不够强** → 问题和解决的视觉差异要明显
5. **情感弧线断裂** → 从问题到解决必须流畅过渡
6. **产品位置边缘化** → 产品必须是"转折点"，不是配角
