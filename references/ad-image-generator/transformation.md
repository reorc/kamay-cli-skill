# Transformation 广告 Prompt 架构设计心法

> 核心理念：**展示"使用前后"的戏剧性变化**

## 一、策略 定位

Transformation 类型广告的核心目标是**通过展示使用产品前后的巨大变化，让用户渴望这种"蜕变"**。与 Problem-Solution 不同，Transformation 更强调"变化的过程"和"变化后的新状态"。

**关键心智模型**：
- Before → 用户的现状（可以更好）
- After → 用户的理想状态（使用产品后）
- 产品 → 实现蜕变的"魔法"

**情感曲线**：
```
现状认知 → 变化渴望 → 蜕变想象 → 行动冲动
(Before)    (Desire)    (After)     (Action)
```

---

## 二、与 Problem-Solution 的区别

| 维度 | Problem-Solution | Transformation |
|------|------------------|----------------|
| 起点 | 痛点/问题 | 现状（不一定是问题） |
| 终点 | 问题解决 | 理想状态/升级版自我 |
| 情感 | 解脱/满足 | 渴望/向往 |
| 叙事 | "摆脱痛苦" | "变得更好" |
| 视觉 | 问题→解决 | Before→After |

**选择指南**：
- 用户有明确痛点 → Problem-Solution
- 用户想要"升级" → **Transformation**

---

## 三、标准章节结构

### 3.1 核心章节

| 章节 | 核心作用 |
|------|----------|
| BEFORE STATE | 使用前的状态展示 |
| TRANSFORMATION MOMENT | 变化发生的瞬间 |
| AFTER STATE | 使用后的理想状态 |
| VISUAL JOURNEY | 变化过程的视觉化 |

### 3.2 辅助章节

| 章节 | 使用场景 |
|------|----------|
| HEADLINE TREATMENT | 变化式文案 |
| PRODUCT AS CATALYST | 产品作为"催化剂" |
| EMOTIONAL ARC | 情感变化曲线 |

## 四、章节设计套路

### 4.1 BEFORE STATE（使用前状态）

**设计原则**：展示"可以更好"的现状，而非"很糟糕"的问题

**视觉化套路**：

1. **普通/平凡**
   ```
   日常场景，没有特别之处
   暗示"这是大多数人的状态"
   → 用户认同："这就是我"
   ```

2. **略显不足**
   ```
   温度在下降、便利性不够
   但不是"痛苦"，而是"可以更好"
   → 用户思考："确实可以改进"
   ```

3. **色调暗示**
   ```
   稍微偏灰或偏冷的色调
   不是"糟糕"，而是"普通"
   → 视觉上的"升级空间"
   ```

**示例模板**：
```
BEFORE STATE:
- Scene: [ordinary daily scenario]
- Visual tone: [slightly muted/ordinary]
- Emotion: [acceptable but improvable]
- Message: "This is where most people are"
- NOT showing: problems or pain points
```

### 4.2 TRANSFORMATION MOMENT（变化瞬间）

**设计原则**：产品是实现变化的"魔法时刻"

**视觉化套路**：

1. **产品登场**
   ```
   产品以动态方式进入画面
   成为"变化的催化剂"
   → 产品是"魔法棒"
   ```

2. **能量爆发**
   ```
   从产品向外辐射的光芒或能量
   暗示"变化正在发生"
   → 视觉化的"蜕变"
   ```

3. **时间线断点**
   ```
   在 Before 和 After 之间
   产品是"分水岭"
   → 产品是"转折点"
   ```

**示例模板**：
```
TRANSFORMATION MOMENT:
- Product entrance: [dynamic/magical/pivotal]
- Visual effect: [energy burst/glow/transition]
- Position: [between before and after]
- Message: "This is where change happens"
```

### 4.3 AFTER STATE（使用后状态）

**设计原则**：展示用户渴望成为的"理想状态"

**视觉化套路**：

1. **升级版生活**
   ```
   更美好的场景、更自由的状态
   暗示"使用产品后的生活"
   → 用户渴望："我想要这样"
   ```

2. **色调升级**
   ```
   明亮、温暖、清新的色调
   与 Before 形成对比
   → 视觉上的"升级感"
   ```

3. **情感升华**
   ```
   满足、自信、自由的情感
   通过人物表情或场景传递
   → 情感层面的"蜕变"
   ```

**示例模板**：
```
AFTER STATE:
- Scene: [upgraded/ideal scenario]
- Visual tone: [bright/warm/aspirational]
- Emotion: [satisfaction/freedom/confidence]
- Message: "This is who you become"
- Product shown: [naturally integrated into ideal life]
```

### 4.4 VISUAL JOURNEY（视觉旅程）

**设计原则**：让用户"看到"变化的过程

**构图套路**：

1. **时间线构图**
   ```
   从左到右或从上到下
   展示 Before → Product → After
   → 线性的变化叙事
   ```

2. **分割对比构图**
   ```
   画面分割展示 Before 和 After
   产品位于分割线
   → 直观的对比
   ```

3. **渐变过渡构图**
   ```
   色调、场景从 Before 渐变到 After
   产品是渐变的"中心"
   → 流畅的变化过程
   ```

**示例模板**：
```
VISUAL JOURNEY:
- Structure: [Timeline/Split/Gradient]
- Flow: Before → Product → After
- Transition: [how change is visualized]
- Eye path: [how viewer experiences the journey]
```

## 五、Transformation 文案套路

### 变化式文案

```
"从普通到非凡"
"升级你的日常"
"遇见更好的自己"
"改变，从一杯开始"
```

### 对比式文案

```
"曾经...现在..."
"以前...从此..."
"告别平凡，拥抱卓越"
```

### 邀请式文案

```
"准备好改变了吗？"
"你的升级时刻"
"开启新的可能"
```

## 七、使用场景

### 适合 Transformation 的情况

- ✅ 产品能带来明显的"升级"体验
- ✅ 目标用户想要"变得更好"
- ✅ 产品有"改变生活"的潜力
- ✅ 需要激发用户的"渴望"情感

### 不适合 Transformation 的情况

- ❌ 产品主要解决具体问题（用 Problem-Solution）
- ❌ 需要强调功能参数（用 Features-Benefits）
- ❌ 需要建立信任（用 Testimonial）

---

## 八、避免的常见错误

1. **Before 太负面** → 应该是"普通"而非"糟糕"
2. **After 太夸张** → 保持可信的"升级"
3. **变化不够明显** → Before 和 After 对比要清晰
4. **产品角色模糊** → 产品必须是"变化的催化剂"
5. **缺乏情感连接** → 用户要能"代入"这个变化
