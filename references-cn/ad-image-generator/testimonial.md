# Testimonial 广告 Prompt 架构设计心法

> 核心理念：**真实的声音，可信的证明**

## 一、策略 定位

Testimonial 类型广告的核心目标是**通过真实用户的声音建立信任**，让潜在用户看到"和我一样的人"已经验证了产品价值。

**关键心智模型**：
- 产品 → 被验证的选择
- 功能 → 真实体验的结果
- 用户 → 可信赖的证人

**信任建立公式**：
```
真实用户 + 具体体验 + 情感共鸣 = 购买信心
```

## 三、章节设计套路

### 3.1 UNEXPECTED ELEMENT（意外元素）

**设计原则**：打破"用户头像+引号"的传统证言广告模式

**Testimonial 特有套路**：

1. **动态产品证言**
   ```
   产品处于动态状态（倾斜、悬浮）
   证言文字环绕产品
   → 产品"活着"证明自己
   ```

2. **时间可视化证言**
   ```
   温度计/时间轴融入设计
   展示"24小时后依然..."的视觉证明
   → 数据化的用户体验
   ```

3. **场景化证言**
   ```
   用户在真实使用场景中
   证言自然融入场景
   → 不是"说"而是"展示"
   ```

4. **对比式证言**
   ```
   展示使用前后的对比
   用户证言解释这个变化
   → 可视化的效果证明
   ```

**示例模板**：
```
UNEXPECTED ELEMENT:
- Break from traditional testimonial format by [specific technique]
- Product shown [dynamic state] while testimonial [integration method]
- Creates authenticity through [visual approach]
- Viewers see PROOF, not just CLAIMS
```

### 3.2 COMPOSITION & EMOTIONAL STORY（构图与情感叙事）

**设计原则**：构建"真实用户的真实故事"

**常用套路**：

1. **三段式叙事**
   ```
   上部：用户证言/故事
   中部：产品在使用中
   下部：功能证明/数据
   → 故事-证据-结论
   ```

2. **亲密视角构图**
   ```
   近距离展示手持产品
   证言文字作为"内心独白"
   → 代入感和真实感
   ```

3. **时间线构图**
   ```
   展示用户一天中使用产品的多个时刻
   证言贯穿时间线
   → "全天候"的真实体验
   ```

**示例模板**：
```
COMPOSITION & EMOTIONAL STORY:
- Structure: [Three-part/Intimate/Timeline]
- Testimonial position: [integrated naturally, not overlaid]
- Product context: [real use scenario]
- Emotional arc: [problem] → [discovery] → [satisfaction]
- Authenticity cues: [specific elements that feel real]
```

### 3.3 CUSTOMER TESTIMONIAL（用户证言）

**设计原则**：证言必须具体、可信、有情感

**证言写作套路**：

1. **具体体验型**
   ```
   "我的茶，从早到晚都完美"
   "24小时后打开，还是热的"
   → 具体的使用场景和结果
   ```

2. **问题解决型**
   ```
   "终于不用担心茶凉了"
   "再也不需要反复加热"
   → 解决了什么具体问题
   ```

3. **情感表达型**
   ```
   "这不是杯子，是我忙碌生活的缓冲剂"
   "每一口都是对自己的善待"
   → 情感层面的价值
   ```

4. **对比发现型**
   ```
   "用过才知道，之前的杯子都是将就"
   "原来保温可以做到这种程度"
   → 对比后的惊喜发现
   ```

**证言格式**：
```
"[具体体验/情感表达]"
—— [用户名]，[城市]
⭐⭐⭐⭐⭐
```

**示例模板**：
```
CUSTOMER TESTIMONIAL:
- Quote type: [Specific Experience/Problem Solved/Emotional/Discovery]
- Content: "[具体的中文证言]"
- Attribution: [Name], [City]
- Rating: 5-star visual
- Tone: Conversational, authentic, NOT corporate
```

### 3.4 PRODUCT PRESENTATION（产品展示）

**设计原则**：产品必须在"被使用"的状态中展示

**常用套路**：

1. **手持展示**
   ```
   真实的手自然握持产品
   展示日常使用的瞬间
   → 真实感和尺度感
   ```

2. **场景融入**
   ```
   产品在真实环境中（办公桌、户外、通勤）
   不是摄影棚的完美布景
   → 生活化的真实感
   ```

3. **功能可见**
   ```
   展示产品正在"工作"
   （蒸汽可见、茶叶浸泡、温度保持）
   → 功能的视觉证明
   ```

**示例模板**：
```
PRODUCT PRESENTATION:
- Context: [real-life scenario]
- Human element: [hands holding/interacting]
- Function visible: [steam/condensation/tea brewing]
- NOT a studio shot - feels like [real moment captured]
- Product is being USED, not just displayed
```

### 3.5 VISUAL DISTINCTIVENESS（视觉独特性）

**设计原则**：在真实感和记忆点之间找到平衡

**常用套路**：

1. **动态角度**
   ```
   产品倾斜或非常规角度
   打破静态证言广告的沉闷
   → 能量感 + 真实感
   ```

2. **光线真实感**
   ```
   自然光线或生活化光线
   不是过度修饰的商业光线
   → 可信的真实感
   ```

3. **证言视觉化**
   ```
   将证言内容转化为视觉元素
   （温度计显示"24小时"、时间线展示体验）
   → 证言不只是文字
   ```

**示例模板**：
```
VISUAL DISTINCTIVENESS:
- Memorable element: [specific visual that stands out]
- Authenticity balance: [real enough to trust, distinctive enough to remember]
- Testimonial visualization: [how words become visuals]
- Breaks from typical testimonial ads by [specific difference]
```

### 3.6 AVOID（避免事项）

**Testimonial 特别需要避免的错误**：

1. **过度完美**
   ```
   ❌ 过于精致的摄影棚照片
   ✅ 有生活感的真实场景
   ```

2. **虚假感**
   ```
   ❌ 过于夸张的证言
   ✅ 具体、可信的体验描述
   ```

3. **缺乏人味**
   ```
   ❌ 只有产品没有人
   ✅ 人与产品的真实互动
   ```

4. **证言悬浮**
   ```
   ❌ 证言文字简单覆盖在图片上
   ✅ 证言融入构图设计
   ```

**示例模板**：
```
AVOID:
- Over-polished studio photography (feels fake)
- Generic testimonial quotes (not specific enough)
- Product without human element (lacks authenticity)
- Text simply overlaid (not integrated)
- Corporate tone in testimonial (should feel personal)
```

## 四、证言文案模板库

### 具体体验型

```

```

```

"每一口都是对自己的善待"
"温度恒定，就像有人在乎你"
```

```
"用过才知道，之前都是将就"
"原来保温可以做到这种程度"
"没想到真的只降2度"
"第一次相信广告说的是真的"
```

## 六、信任建立检查清单

在完成 Testimonial Prompt 后，检查以下要素：

- [ ] 证言是否具体？（有具体场景、时间、感受）
- [ ] 是否有人的元素？（手、使用场景）
- [ ] 产品是否在"被使用"状态？
- [ ] 视觉是否有真实感？（不过度精致）
- [ ] 证言是否融入设计？（不是简单覆盖）
- [ ] 是否有可信的归属？（用户名、城市）
- [ ] 整体调性是否亲近？（不是企业宣传感）

---

## 七、避免的常见错误

1. **证言太泛泛** → 必须有具体的体验细节
2. **过度美化** → 保持真实感，不要太"广告"
3. **缺乏人味** → 必须有人的元素（手、场景）
4. **证言悬浮** → 文字必须融入设计
5. **调性太商业** → 应该像朋友推荐，不是企业宣传
6. **忽略视觉证明** → 证言内容要有视觉对应
