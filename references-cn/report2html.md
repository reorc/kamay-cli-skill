# 报告转 HTML (Report to HTML)

## 简介

报告转 HTML 是一个将数据分析报告转换为专业 HTML 页面的能力。它可以生成响应式、自包含的单页 HTML 文件，适用于品类分析、黑马分析、VOC 分析等各类报告场景。

## 使用场景

- **报告分享**：需要将分析报告以网页形式分享给团队成员或客户
- **专业展示**：需要将数据以可视化、美观的方式呈现
- **报告存档**：生成可独立打开的 HTML 文件，便于长期保存
- **汇报演示**：在会议或演示中展示分析结果

## 核心能力

### 数据保真与格式化
- 严格遵循原始数据，不捏造任何指标或洞察
- 数字格式规范：千位分隔符、百分比保留1-2位小数、货币符号
- 表格数字右对齐，文本左对齐

### 专业视觉设计
- **可读性优先**：清晰的标题层级、适当的间距、舒适的行高
- **克制的配色**：简约统一的配色方案，颜色服务于内容理解
- **视觉层次**：关键指标突出显示，辅助信息适度弱化
- **响应式布局**：适配桌面、平板、手机等不同屏幕尺寸
- **微妙交互**：悬停效果和过渡动画

### 数据可视化
根据数据类型选择最有效的可视化方式：

| 数据类型 | 推荐图表 |
|---------|---------|
| 对比/排名/市场份额 | 横向条形图、饼图/环形图 |
| 时间序列/趋势 | 折线图、面积图、分组柱状图 |
| 关键指标 | 大数字卡片 + 趋势指示器（↑↓） |
| 分布 | 堆叠柱状图、树形图、热力图 |

### 支持的报告类型
- 品类分析报告
- 黑马分析报告
- VOC（用户之声）分析报告
- 自定义报告

## 如何使用

### 基本用法

当你有数据分析报告需要转换为 HTML 时，直接告诉 Kamay：

```
将这份报告转换为 HTML 页面
```

```
把这个品类分析报告生成一个专业的 HTML 报告
```

### 使用流程

1. **提供报告内容**：将分析报告的内容（数据、表格、洞察等）提供给 Kamay
2. **指定报告类型**：说明是品类分析、黑马分析、VOC 分析还是其他类型
3. **生成 HTML**：系统会自动设计布局并生成 HTML 文件
4. **获取链接**：生成完成后获得可分享的报告链接

### 输出文件

- **文件命名**：`{报告类型}_{日期}.html`，如 `Category_Analysis_2025-01-19.html`
- **文件编码**：UTF-8
- **自包含**：所有样式内联，无需外部依赖
- **响应式**：适配各种屏幕尺寸

### 设计参考

报告生成时会遵循以下设计原则：

**颜色语义**：
- 绿色 ↑ 表示增长
- 红色 ↓ 表示下降

**布局原则**：
- 摘要在前，关键指标突出
- 支持性内容按层级递进
- 图表与表格合理搭配

## 质量检查

生成的报告会自动进行以下检查：

**数据检查**：
- 数字格式正确（千位分隔符、货币符号）
- 百分比保留合理小数位
- 表格对齐正确
- 数据无错误

**设计检查**：
- 视觉层次清晰
- 配色统一协调
- 不同屏幕尺寸可读
- 响应式布局正常

**内容检查**：
- 所有内容来自原始数据
- 无虚构或捏造的指标
- 语义化 HTML5 结构

## 示例

### 输入
```

- 市场规模：$12,345,678

## 品类趋势

- 天然成分产品需求上升
...
```

### 输出
系统会生成一个专业的 HTML 报告页面，包含：
- 响应式布局
- 数据可视化图表
- 格式化的数字和表格
- 可分享的链接
